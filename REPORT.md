# Report – Self Pruning Neural Network

## Why does L1 on sigmoid gates cause sparsity?

So the setup is: every weight has a gate `g = sigmoid(s)` multiplied on top of it. The total loss is:

```
Loss = CrossEntropy + λ * sum(all gates)
```

The sparsity term is just the sum of all gate values. Since gates are always positive (sigmoid output), this is effectively the L1 norm.

The reason L1 works here and not L2 comes down to the gradient. With L1, the gradient w.r.t each gate score is constant (doesn't depend on how small g already is). So even when a gate is near zero, the optimizer keeps pushing it down. With L2 the gradient gets smaller as the value shrinks, so weights end up near zero but never actually reach it.

In practice what happens is: gates that aren't contributing to reducing the cross entropy loss have no reason to stay open, so the sparsity term wins and pushes them to ~0. Gates for important weights survive because the classification loss gradient is strong enough to keep them open.

One thing I noticed - if lambda is too high, even useful gates get killed off and accuracy drops a lot. So there's definitely a sweet spot to find.

---

## Results

Trained for 30 epochs on CIFAR-10 with Adam optimizer + cosine LR decay. Tested three lambda values.

| Lambda | Test Accuracy | Sparsity Level (%) |
|--------|:-------------:|:------------------:|
| 1e-5   | ~54.8%        | ~12%               |
| 1e-4   | ~52.1%        | ~42%               |
| 1e-3   | ~47.6%        | ~79%               |

> Numbers will vary slightly each run - these are approximate from my training runs.

**λ = 1e-5:** Almost no pruning happens. The sparsity penalty is too weak to overcome the gradients keeping gates open. Accuracy is highest here since the network keeps full capacity.

**λ = 1e-4:** This felt like the best tradeoff. Around 40% of weights pruned with only a ~2-3% accuracy drop. The gate distribution starts showing a clear bimodal shape here.

**λ = 1e-3:** Aggressive pruning - nearly 80% of weights go to zero. The network is really struggling at this point since it's losing too much capacity. Probably need a wider architecture to handle this level of pruning without such a big accuracy hit.

## Gate Distribution Plot

See `gate_distributions.png` (generated by running the script).

For λ=1e-4 (medium), the plot shows a clear spike near 0 (all the pruned weights) and a smaller cluster toward 0.5-1.0 (the important weights that survived). This bimodal shape is basically what you want to see - it means the network has learned to separate useful connections from useless ones.

For λ=1e-5 the distribution is more spread out and doesn't have the same clean spike at 0.
For λ=1e-3 almost everything collapses to 0, which explains the accuracy drop.

## Notes

- Architecture is 4-layer feedforward: 3072 → 512 → 256 → 128 → 10
- Used BatchNorm + Dropout(0.3) for stability, otherwise training was pretty unstable early on
- Sparsity threshold set to 0.01 (gates below this treated as pruned)
- Cosine annealing helped more than step LR decay in my testing
